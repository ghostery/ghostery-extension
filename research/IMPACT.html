Across two regions and 21 publishers, AI agents pay substantially less per page-load when their browser blocks ads and dismisses consent dialogs. The savings are bigger than typical adblock benchmarks suggest — because every agent turn re-sends the whole prior conversation, dismissal cycles cost much more than the per-turn token count alone.

Where the cost actually comes from

Picture one page — wp.pl. The agent's job: "find the top headline." Its bill breaks into four pieces.

1
The size of one look. Whatever the agent reads each turn — the HTML, just the visible text, an accessibility outline, or a screenshot. Different agents use different things, so we measure all four. Sizes range from a few hundred tokens (plain text) to over a million (raw HTML).
2
The overhead of one look. A single round-trip to the model: send the page, get back the next action. Around 3 000 – 5 000 tokens once you add the system prompt, the image and a slice of page text.
3
How many looks the task takes. The model has no memory between calls — every new turn re-sends everything from earlier turns. A 5-look task isn't 5× a 1-look task; it's closer to 15×, because turn 5 re-reads turns 1 – 4. When a banner blocks the page, the agent burns 4 extra looks finding and clicking dismiss, each one re-paying for every prior look. Almost all of Ghostery's saving comes from here.
4
What the model writes back. If the agent asks the model to describe what's on screen, a cluttered page produces more output. A clean Ghostery page produces less, usually — though on US sites it sometimes produces more because the clean page surfaces more articles for the model to list. Small effect either way.

Bill per page-load ≈ (size of one look) × (how many looks) + (whatever the model writes back). Ghostery does almost nothing to size — pages don't shrink. It dramatically cuts how many looks by eliminating the banner-dismissal loop. That's the whole story; everything else is a rounding effect.

What this means

If you ship an AI agent that browses the web — research-assistants, deal-hunters, knowledge-workers, web-scrapers — the cost reduction from running Ghostery inside the browser is bigger than a typical "blocked ad bytes" measurement would suggest. The dominant saving is the agent not having to dismiss the consent banner, and that saving compounds every turn the model would otherwise spend looking at, deciding about, and clicking dismiss buttons.

The effect is strongest where consent dialogs are most common (EU traffic) and shrinks proportionally on traffic mixes where most pages don't show a CMP to a headless browser. Vision-mode agents and innerText agents see the biggest percentage gains because their per-turn cost is moderate; full-HTML scrapers see the smallest because their per-turn cost is dominated by the page they were going to load anyway.

Method: Per page × variant runs in a fresh Chrome for Testing session with anti-bot fingerprints suppressed. 5 samples per cell, median reported. The "how many looks" number comes from Anthropic's computer-use API on Sonnet 4.5; the "viewport cost" number comes from asking Sonnet 4.5 to inventory each viewport screenshot.

Caveats: 21 pages across two regions can't support an industry-wide per-million-loads claim. Visibility detector misses modals rendered in shadow DOM (Ketch, OneTrust 2.x, Guardian custom CMP) — those modals are real and confirmed by screenshots; the autoconsent oracle catches some but not all. Anti-bot-blocked pages (dailymail, allrecipes, reuters, imdb, people, politico) are excluded.
